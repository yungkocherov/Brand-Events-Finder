# Brand Events Finder

Веб-сервис для поиска значимых событий по брендам — ребрендингов, скандалов, ухода с рынка, новых продуктов и т.д. AI-агент фильтрует результаты и оценивает влияние каждого события на бизнес-метрики (1-5). Результаты можно выгрузить в CSV с dummy-переменными для использования в моделях.

## Что умеет

- Поиск событий по списку брендов одновременно
- 13 категорий событий (уход с рынка, ребрендинг, новые продукты, санкции, скандалы, сделки, фарма-регистрация и т.д.)
- Свои поисковые запросы (например, "отзыв лицензии")
- Фильтр по отрасли — отсеивает однофамильцев из других сфер
- AI-фильтрация через Mistral (бесплатный API)
- Оценка влияния события на метрики (1-5 звёзд)
- Тональность (позитивное / негативное / нейтральное)
- Таймлайн событий
- Фильтры по категории, сортировка по влиянию/дате
- Экспорт в CSV: dummy-переменные (0/1) по дням/неделям/месяцам
- Предпросмотр CSV перед скачиванием
- Возможность исключить события из CSV вручную
- История поисков (хранится в браузере)

## Установка

### Требования
- Python 3.10+
- Доступ к Mistral API (через VPN из РФ)

### Шаги

1. Клонировать репозиторий:
```bash
git clone https://github.com/yungkocherov/brand_events.git
cd brand_events
```

2. Запустить сервис:
```bash
start.bat
```

При первом запуске скрипт автоматически создаст виртуальное окружение и установит зависимости. Сервер запустится на http://localhost:8080

### Ручной запуск (без start.bat)
```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

## Использование

1. Открой http://localhost:8080
2. Получи бесплатный API-ключ на https://console.mistral.ai (регистрация занимает минуту)
3. Вставь ключ в поле "Mistral API Key" и нажми "Проверить"
4. Введи бренды через запятую: `Нурофен, Терафлю, Спазмалгон`
5. Укажи отрасль: `фармацевтика`
6. Выбери типы событий или добавь свои
7. Нажми "Найти события" — поиск занимает ~10-30 секунд на бренд
8. Просмотри результаты, отметь нужные события чекбоксами
9. Скачай CSV для использования в моделях

### CSV формат

Колонки: `date, brand1__event_name, brand2__event_name, ...`

Каждая колонка — dummy-переменная: `0` до даты события, `1` начиная с даты события. Имена колонок в snake_case с транслитерацией.

Пример:
```
date,nurofen__povyshenie_cen,teraflyu__zapusk_novoy_lineyki
2024-01-01,0,0
2024-02-01,0,1
2024-03-01,1,1
```

```
├── models.py            Pydantic-модели запросов и ответов
├── services/
│   └── event_search.py  Логика поиска: DDG + Mistral
└── static/
    └── index.html       Single-page фронтенд (vanilla JS)
```

### Поток данных

1. **Frontend** → POST `/api/search` с брендами и параметрами
2. **DuckDuckGo Search** — отдельный запрос на каждый тип события (макс 10 результатов)
3. **Дедупликация** по URL
4. **Mistral AI** — фильтрует мусор, оставляет реальные события про бренд, оценивает impact_score 1-5
5. **3 попытки** при сбое Mistral, фоллбэк на сырые результаты
6. **Сортировка** по impact_score (по убыванию)
7. **Frontend** → отображает с фильтрами и сортировкой

## Безопасность

- API-ключ Mistral хранится **только в браузере** (localStorage)
- Бэкенд получает ключ из тела запроса и не сохраняет
- Никаких ключей в `.env` на сервере
- Сервис работает локально, не требует VPS

## Ограничения

- DuckDuckGo может блокировать при частых запросах — добавлены паузы 0.5 сек
- Mistral API недоступен из РФ без VPN
- Бесплатный tier Mistral: 500 000 токенов/день
- Точность зависит от качества промпта и качества выдачи DDG

## Стек

- **Backend**: FastAPI, Pydantic, httpx
- **Search**: ddgs (DuckDuckGo)
- **AI**: mistralai
- **Frontend**: HTML + vanilla JavaScript (без фреймворков)
